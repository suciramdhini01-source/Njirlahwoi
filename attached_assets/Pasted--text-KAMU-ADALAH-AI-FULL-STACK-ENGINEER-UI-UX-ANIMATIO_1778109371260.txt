## 1. DESKRIPSI APLIKASI
NJIRLAH AI adalah platform chat multi‑model AI dengan dua sumber:
- **OpenRouter** → WAJIB BYOK (pengguna harus memasukkan API key miliknya sendiri, termasuk yang gratis). Tanpa key, pengguna tidak bisa mengakses model OpenRouter.
- **Cloudflare Workers AI** → Token API disediakan server (bukan BYOK). Selalu bisa digunakan tanpa key tambahan.

## 3. ARSITEKTUR BACKEND (PROXY)
### Proxy OpenRouter (BYOK strict)
- Endpoint: `POST /api/openrouter/chat`
- Meneruskan header `x-api-key` dari request ke `https://openrouter.ai/api/v1/chat/completions` dengan header `Authorization: Bearer {key}`.
- **WAJIB streaming (SSE)**. Gunakan `fetch` dan teruskan `ReadableStream` ke respons.
- **TIDAK BOLEH menyimpan key di server**. Hanya proxy transparan.

### Proxy Cloudflare (built‑in token server)
- Endpoint: `POST /api/cloudflare/chat`
- Gunakan environment variable `CLOUDFLARE_API_TOKEN` dan `CLOUDFLARE_ACCOUNT_ID`.
- Panggil `https://api.cloudflare.com/client/v4/accounts/{ACCOUNT_ID}/ai/run/{model}` dengan format body Cloudflare Workers AI (`{ messages: [...] }`).
- **WAJIB streaming** juga.

## 4. ENKRIPSI KUNCI OPENROUTER (CLIENT‑SIDE)
