## 1a) Updated prompt (versi diperjelas & dipadatkan)

Kamu adalah **Senior UI/UX Engineer** sekaligus **Frontend Architect**. Tugasmu adalah merancang **major UI/UX overhaul** untuk aplikasi **NJIRLAH AI** agar terasa **premium, intuitif, solid, dan menyenangkan** setara aplikasi kelas dunia—tanpa mengubah perilaku backend (perubahan hanya UI/UX + integrasi state/store).

### Prinsip utama (3 pilar)
1. **UX Intuitif**: alur kerja jelas, minim friksi, affordance kuat.
2. **UI Premium**: dark mode mendalam, glassmorphism, animasi halus, tipografi rapi, aksen neon.
3. **Platform Solid**: dashboard untuk manajemen API key, health/status, dan full responsive.

---

## A. Perubahan struktural halaman

### A1) Contextual Chat Header
- Header menampilkan **judul chat aktif** (bukan label statis).
- Jika belum ada judul: tampilkan **“Chat Baru”** + ikon edit (rename inline/modal).
- Aksi kanan: **Pin**, **Favorite**, dan menu **⋯** (rename, hapus, ekspor).
- Transisi saat pindah chat: judul **fade + slide dari atas**.

### A2) Sidebar ditingkatkan
- Seksi **Pinned** dan **Favorites** di bagian atas, dengan divider.
- Item pinned/favorite lebih ringkas dan **drag & reorder**.
- **Auto-generated title**: setelah pesan pertama, panggil `/api/generate-title` untuk judul 3–5 kata; tampilkan di sidebar dengan animasi typing halus.
- **Empty state**: ilustrasi unicorn kecil + teks “Mulai percakapan pertamamu” + tombol “Chat Baru” kontras.
- **Collapse/Expand**: sidebar bisa di-drag lebarnya (Framer Motion drag). Saat collapsed: hanya ikon.

### A3) Halaman “API NJIR” jadi dashboard visual (bukan form datar)
- Layout: **grid cards** per provider (OpenRouter, Cloudflare, Alibaba, Custom), tiap kartu memuat:
  - Nama & logo provider
  - Status koneksi terakhir (indikator hijau/merah berdenyut)
  - Latensi rata-rata (diukur saat test connection)
  - Jumlah model tersedia
  - Tombol **Kelola** → buka modal konfigurasi
- Bagian atas: **mini bar chart** perbandingan latensi antar provider (CSS animasi atau library ringan).
- Empty state: kartu kosong + ikon kunci + “Belum ada penyedia terhubung” + CTA “Tambah Penyedia Pertama”.
- Tooltip bantuan di setiap field input, termasuk link sumber key (mis. OpenRouter keys URL).

## B. Penyempurnaan komponen utama

### B1) Chat Input & Attachments
- **Smart attachments**: upload gambar/PDF/kode → tampil chip preview (nama, ikon, hapus). File otomatis masuk konteks AI.
- Input lebih kaya: tombol lampiran (kiri), tombol kirim (kanan) dengan animasi morph jadi checkmark saat terkirim.
- Placeholder dinamis sesuai konteks (mis. normal vs ada lampiran).

### B2) Chat bubble & pesan
- Pada hover bubble AI: tombol **Regenerate** + **Copy** + tooltip.
- **Like/Dislike** di bawah bubble AI; klik → animasi pop singkat; simpan di state lokal (tanpa kirim server).
- **Error bubble**: merah transparan + ikon warning + pesan ramah + CTA “Coba Lagi” / “Periksa Kunci API”.

### B3) Model selector lebih kuat
- Search teks realtime.
- Filter: Provider (OpenAI/Anthropic/dll), Tipe (chat/image/code), Status (gratis/berbayar).
- Model item berupa **card**: ikon provider, nama model, badge GRATIS, harga/token kecil.
- Toggle tampilan **list ↔ grid**.

## C. Animasi & mikrointeraksi (wajib)
- Skeleton loading shimmer untuk model list / chat history.
- Skeleton bubble saat menunggu respons AI pertama.
- Toast notifikasi muncul dari kanan atas dengan spring/bounce ringan.
- Hover scale 1.02x untuk semua kartu (0.2s).
- Progress bar neon cyan tipis di bawah chat header saat streaming.
- Auto-scroll cerdas: auto scroll saat streaming kecuali user scroll ke atas; bila user di atas → tampil tombol “Scroll ke bawah” mengambang.
- Confetti setelah agent selesai generate proyek (canvas-confetti).
